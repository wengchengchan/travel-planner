# Travel Planner MVP

這是旅遊規劃工具的第一版最小可用原型（MVP）。

## 目前內容

- `trip-input.md`：旅遊需求原文
- `trip.json`：結構化行程資料
- `trip.html`：手機可讀的展示頁

## 第一版目標

- 使用者輸入旅遊需求
- 產出文字版行程
- 產出 `trip.json`
- 產出 `trip.html`
- 後續可發佈到 GitHub Pages

## 建議工作流

1. 先整理需求到 `trip-input.md`
2. 依需求生成 `trip.json`
3. 由 `trip.json` 或既有內容生成 `trip.html`
4. 驗證手機上閱讀是否順暢
5. 後續再放上 GitHub Pages

## 第二階段（未做）

- Google Calendar 同步
- 自動發佈到 GitHub Pages
- 真實餐廳/交通/票券資料整合
- 多旅程管理（`trips/`）

## 原則

第一版先求：

- 穩
- 清楚
- 可看
- 可擴充

不要一開始就做成大型平台。
